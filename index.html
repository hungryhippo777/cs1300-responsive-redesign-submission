Hacker News: Responsive Redesign

Attracting 300,000 users per day, Hacker News is a trusted source for computer science, start-ups, technology, and entrepreneurship news. However, its dated and confusing interface begs the question: what intentional design changes can we make to improve upon its flaws?

Identifying Usability Problems

Conceptual Model

How the interface reveals to the user how it works

Four key drivers

  1. Links underline on hover
  2. Clear redirection links to other pages on the header and footer
  3. A standalone "more" button to load additional results
  4. Search bar

Usability

Is it easy to operate and complete the necessary tasks?

No

While the functionality of the site is simple, the buttons are not intuitive or easy to discern amid the cluttered page. Additionally, user sign-in is required for many actions.

Learnability

Is it easy to learn?

Yes and No

Repeatability of the simple tasks leads to recognition, however there are many nuanced features that can only be found in the non-obvious "Guidelines" and "FAQ" links in the footer.

Memorability

Is it easy to recall what to do?

It depends on the task

For external links, redirecting to another vibrant webpage is memorable. However, internal redirections are bland and repetitive from multiple links.

Efficiency

Can typical tasks be done quickly?

No

The biggest issues arise from the necessity to take time to sign in to access many features, and the inability to go back a page when exploring past the first page of 20 articles.

Accessibility

Checked using WebAIM WAVE

Multiple accessibility issues to remediate

  • Low contrast errors
  • Small text alerts
  • Missing form label for the search bar
  • Empty links for upvote icons
  • Missing alternative text for website logo
